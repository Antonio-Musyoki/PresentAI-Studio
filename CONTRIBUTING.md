# Contributing to PresentAI Studio

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Commit: `git commit -am 'Add feature'`
6. Push: `git push origin feature/your-feature`
7. Submit a Pull Request

## Branch Naming

- `feature/` - New features
- `bugfix/` - Bug fixes
- `refactor/` - Code refactoring
- `docs/` - Documentation
- `infra/` - Infrastructure changes

## Commit Messages

Use conventional commits:

```
feat: add new slide layout
fix: resolve PDF parsing issue
docs: update API documentation
refactor: improve design engine
test: add PDF intelligence tests
```

## Code Style

- Use TypeScript
- Follow ESLint rules
- Format with Prettier
- Write meaningful variable/function names
- Add JSDoc comments for public APIs

## Testing

```bash
npm run test
```

## Building

```bash
npm run build
```

## Pull Request Process

1. Ensure all tests pass
2. Update documentation
3. Add descriptive PR title and description
4. Link related issues
5. Request review from maintainers

## Code Review

We review all PRs for:

- Code quality
- Test coverage
- Documentation
- Performance implications
- Security concerns
